## Что это за проект
PROCLUSTER — онлайн-сервис кластерных графиков крипто-активов
(аналог mobchart.com, exocharts, ATAS). Кастомный canvas/WebGL движок графика,
футпринт/кластера/японские свечи, стакан DOM, индикаторы, авторизация, тарифы, админка.
- Лендинг: procluster.online
- Приложение: chart.procluster.online

## Стек (не менять без записи в docs/DECISIONS.md)
- Backend: Go, один бинарник + горутины, WebSocket hub, REST API
- Market data: ClickHouse
- Hot aggregation: Redis
- Auth/settings: SQLite (старт) → план миграции на PostgreSQL
- Frontend: React + TypeScript (strict)
- Chart engine: PixiJS (WebGL) + Canvas2D для текста. НЕ TradingView Lightweight Charts!
- Deploy: Docker Compose на VPS, CI/CD через GitHub Actions
- Данные: Binance WS (trades stream) + историческая загрузка с data.binance.vision

## Железо на старте (учитывать при оптимизации)

## Правило: безопасность (код в auth/**, admin/**, api/**)
- Весь пользовательский ввод валидируется на бэкенде.
- SQL только параметризованный, без конкатенации.
- Приватные/админ-маршруты: authN+authZ на КАЖДОМ запросе.
- login/register/recovery: rate-limit (Redis) + lockout.
- Никаких секретов в коде. Ошибки наружу без stack trace.
- Лимит одновременных графических WS-сессий по userId — на бэкенде, по тарифу (Redis).
  Проверка+регистрация атомарны (Lua). Heartbeat + удаление протухших.
  Политика по умолчанию: last-wins. Клиенту не доверять.
- REST API: /api/v1/candles — единый JSON-контракт {ok, data, error}.
- WebSocket hub: подписка по (symbol, market, timeframe), heartbeat, session limit.
- При сомнении — свериться с docs/SECURITY.md.

## Правило: движок графика (код в frontend/src/chart-engine/**)
- Только типизированные массивы в hot-path, object pool, без аллокаций в render loop.
- Движок НЕ импортирует UI-компоненты напрямую — только через engine.* API/события.
- Цель 60 FPS; перерисовка по dirty-flag, батчинг draw calls.
- Текст — Canvas2D слой; геометрия — WebGL/PixiJS.
- Контракт включает engine.pause() / engine.resume() (пассивная/активная вкладка).
- Детали: docs/CHART_ENGINE.md.

## Правило: агрегация данных (код в aggregator/**, store/**, ingest/**, migrations/**)
- От этого зависит ТОЧНОСТЬ футпринта (цель — близко к ATAS / Tiger Trade).
- ЕДИНЫЙ источник правды для округления/сжатия/маппинга — один модуль, используется везде
  (live, история, стакан). Дублировать запрещено.
- Округление объёма: TRUNCATE до 1 знака (5.1256→5.1; 5.627→5.6; 0.0125→0.0; 0.85→0.8).
- isBuyerMaker==true → ASK/SELL; ==false → BID/BUY. Сортировка по tradeId. Время — trade time.
- priceTick: futures 0.1$, spot 0.01$. base-сжатие из конфигурации символа, НЕ хардкод.
  Уровни = base*k до 10. Объединение рядов из base.
- ТФ: 1m раз в минуту, 1h раз в час и т.д. Минимальный ТФ детально, старшие — rollup.
- ClickHouse: кластеры отдельными строками; таблицы clusters_futures/spot и *_dom со своими TTL.
- Запросы параметризованные. Историческая загрузка идемпотентна.
- При сомнении по новому тикеру — СПРОСИ. Сверься с docs/DATA_MODEL.md и docs/DOM_SPEC.md.

## Правило: Go-стиль (код в backend/**/*.go)
- gofmt/goimports. Один бинарник с горутинами. Логика в internal/ по пакетам.
- Доступ к БД через repository-интерфейсы (для миграции SQLite→PostgreSQL).
- Ошибки оборачивать (fmt.Errorf "...: %w"), не игнорировать. Без panic в рантайме запросов.
- Останов воркеров через context.Context, без утечек горутин. Защита общих данных. Тесты с -race.
- Hot-path (ingest/aggregator/ws-hub): без лишних аллокаций; запись в ClickHouse батчить асинхронно.
- Весь внешний ввод валидируется. Секреты только через env. Логи без паролей/токенов.
- Юнит-тесты на округление, сжатие, агрегацию по ТФ, формулы индикаторов.
- Минимум зависимостей; крупная библиотека — с обоснованием в DECISIONS.md.
