# CHART_ENGINE.md — PROCLUSTER

## Цель
Спецификация кастомного движка графика. Главное требование: МАКСИМАЛЬНАЯ СКОРОСТЬ, 60 FPS.
Аналоги: mobchart.com, app.mmt.gg. Запрещено: TradingView Lightweight Charts.

## Технология
- WebGL через PixiJS — отрисовка свечей, кластеров, объёмов (тысячи прямоугольников/текстур).
- Canvas2D — текст (цифры bid/ask, оси, подписи), т.к. текста много и WebGL-текст дорог.
- Гибрид: WebGL-слой снизу, Canvas2D-слой текста сверху, синхронизированы по трансформации.
- Движок ИЗОЛИРОВАН в frontend/src/chart-engine. Наружу — только API/события
  (setData, updateLastCandle, setCompression, setMode, onViewportChange и т.п.).
  Никаких прямых импортов UI-компонентов внутрь движка.

## Архитектура движка (модули)
- Renderer        — PixiJS app, слои, цикл отрисовки, батчинг
- Viewport        — трансформация координат (time/price ↔ px), зум/пан, инерция
- DataStore       — буфер свечей/кластеров в видимой области + соседние, типизированные массивы
- Scales          — оси X (время) и Y (цена), форматирование
- Renderers/      — candleRenderer, footprintRenderer, clusterRenderer, barRenderer,
                    volumeHistogramRenderer, imbalanceTextRenderer
- Indicators/     — слой для оверлеев и панелей индикаторов (см. INDICATORS.md)
- Interaction     — обработка мыши/клавиатуры (см. ниже)
- Crosshair/Tooltip

## Производительность (правила)
- Только типизированные массивы (Float32Array и т.п.) для геометрии, без объектов на свечу в hot-path.
- Батчинг: один draw call на тип примитива, где возможно (ParticleContainer / Mesh).
- Объекты переиспользуются (object pool), без аллокаций в render loop.
- Перерисовка только при изменении (dirty-flag), не каждый кадр без нужды.
- Текст Canvas2D: рисовать только видимые подписи, кэшировать метрики, не рисовать при сильном отдалении.
- Цель: 60 FPS при панорамировании/зуме на типичном датасете видимой области.

## Типы отображения (режимы свечей)
Переключаются в верхней панели: Авто | Японские свечи | Бары | Футпринт | Кластера.

### Авто-переключение по числу видимых свечей
(стартовые пороги — подтверждены заказчиком)
- < 100 свечей в видимой области → КЛАСТЕРА (bid/ask цифрами)
- 100–300 → ФУТПРИНТ + гистограмма объёма
- > 300 → ЯПОНСКИЕ СВЕЧИ
(Пороги вынести в конфиг движка, чтобы менять без правки кода.)

## Кластер / Футпринт визуал
- Кластер: внутри тела свечи строки ценовых рядов с цифрами bid и ask.
- Отображение объёма в ячейке переключается: Bid×Ask | Volume | Delta.
- Гистограмма объёмов: для режимов кластера и футпринта (горизонтальные бары по объёму ряда).
- Раскраска имбаланса: при диагональном имбалансе > 300%
  ask красить зелёным, bid — красным (свериться с DATA_MODEL.md, направление имбаланса
  диагональное: ask[price] vs bid[price-1]).

## Палитра свечей (настройка)
Переключатель: (красные/зелёные) или (белые/тёмно-серые). Применяется ко всем режимам.
Тема dark/light, стиль liquid glass — берётся из дизайн-репозитория.

## Сжатие кластеров (UI + движок)
- Переключатель уровней сжатия в верхней панели.
- Доступные уровни = base, base*2 ... до 10 уровней (base из конфигурации символа).
- Для уровней выше base объединение рядов: на лету в движке ИЛИ из Redis-кэша с бэкенда.
  Рекомендация: объединение из base-данных на клиенте для отзывчивости + кэш на бэке для тяжёлых.
- По умолчанию для каждого ТФ — уровень сжатия из админки.

## Управление (мышь/клавиатура)
- Свободное перемещение графика (drag).
- Колесо мыши: zoom in/out К ТОЧКЕ ПОД КУРСОРОМ (anchor на позиции мыши).
- SHIFT + колесо → растяжение по вертикали (масштаб цены).
- CTRL + колесо → растяжение по горизонтали (масштаб времени).
- Отдельный zoom по оси Y (цена) и по оси X (время) — drag по осям.
- Автоподстройка цены (autoscale) по видимой области с возможностью отключить.

## Рабочие пространства (layouts)
- Кнопка выбора: 1 график | 2 графика по горизонтали | 2 по вертикали.
- При двух графиках — перетаскиваемый разделитель меняет соотношение размеров.
- График автоматически заполняет всю свободную область экрана.
- Если стакан свёрнут в край — график растягивается до края экрана.

## Данные и загрузка
- При открытии — последние 700 свечей из кэша API.
- Скролл влево (в прошлое) → подгрузка из API по запросу, бесшовно (без рывков вьюпорта).
- Live: обновление последней свечи/кластера по WS каждые 100–500 мс.

## Интерфейс движка (черновой контракт, уточнить при реализации)
- engine.setSymbol(symbol, market, timeframe)
- engine.setData(candles[])              // bulk
- engine.prependData(candles[])          // история при скролле
- engine.updateLast(cluster)             // live
- engine.setMode('auto'|'candle'|'bar'|'footprint'|'cluster')
- engine.setCompression(level)
- engine.setVolumeMode('bidask'|'volume'|'delta')
- engine.setPalette('rg'|'wg')
- engine.on('viewportChange', cb)        // для синхронизации, подгрузки истории
- engine.on('needHistory', cb)           // запрос догрузки
- engine.destroy()
- engine.pause()    // остановить render loop, освободить ресурсы (пассивная вкладка)
- engine.resume()   // возобновить рендер (стала активной)
