# INDICATORS.md — PROCLUSTER

## Цель
Система индикаторов + первый индикатор Cluster Search (аналог ATAS).

## Архитектура индикаторов
- Индикатор = отдельный модуль/скрипт с единым интерфейсом.
- Размещение: оверлей на графике ИЛИ в отдельной панели на графике.
- Серверная часть индикатора (если нужны тяжёлые расчёты) — в backend/internal/indicators.
- Клиентская часть — рендер через слой Indicators движка (см. CHART_ENGINE.md).
- Пресеты настроек индикаторов сохраняются (SQLite, привязка к пользователю) и кэшируются (Redis).
- Набор индикаторов по умолчанию и их пресеты задаются в админке per символ/рынок/ТФ.

## Интерфейс индикатора (черновой контракт)
- id, name, type ('overlay' | 'panel')
- defaultSettings
- compute(visibleClusters, settings) → результат для рендера
- render(ctx/pixiLayer, computed, viewport)
- settingsSchema (для авто-генерации UI настроек)

## Cluster Search (первый индикатор, добавлен по умолчанию для ВСЕХ типов)
Логика близка к ATAS. Подсвечивает ценовые ряды/зоны, удовлетворяющие фильтрам.

### Два фильтра: Средний и Крупный (у каждого вкл/выкл)

**Средний фильтр:**
- мин объём (значение), макс объём (значение)
- одно выделение в баре — чекбокс вкл/выкл
- объединение уровней цены (значение, по умолчанию 3)
- перевес по bid/ask (% , по умолчанию 60)
- дельта (опц. условие)
- расположение в свече: любое | тело | нижняя тень | верхняя тень (по умолчанию: любое)
- уведомление в ТГ (только VIP/Admin), опционально

**Крупный фильтр:**
- мин объём (значение)  [у крупного нет макс]
- одно выделение в баре — чекбокс вкл/выкл
- объединение уровней цены (по умолчанию 3)
- перевес по bid/ask (по умолчанию 60%)
- дельта
- расположение в свече (по умолчанию: любое)
- уведомление в ТГ (только VIP/Admin), опционально

### Визуал для каждого фильтра (раздельно)
- мин размер выделения
- макс размер выделения
- форма: круг | квадрат | ромб
- цвет выделения для bid-имбаланса и отдельно для ask-имбаланса
- прозрачность

### Логика расчёта (ТОЧНО)
Для каждой группы из `объединение уровней` подряд идущих ценовых рядов:
1. total_volume = сумма (bid+ask) по этим рядам.
2. Проверка фильтра объёма:
   - Средний: min ≤ total_volume ≤ max
   - Крупный: total_volume ≥ min
3. Считаем имбаланс по группе:
   - bid_imbalance% = (sum_bid / total_volume) * 100
   - ask_imbalance% = (sum_ask / total_volume) * 100
4. Перевес: пользователь вводит ОДНО положительное число (например 60).
   Условие выполнено, если max(bid_imbalance, ask_imbalance) > порог.
   Доминирующая сторона = та, у которой % больше → выбирает цвет (bid-цвет или ask-цвет).
5. Доп. условие дельты — если задано (например delta > X или знак дельты), применяем.
6. Расположение в свече — фильтр по тому, где находится группа рядов:
   тело / нижняя тень / верхняя тень / любое.
7. Если "одно выделение в баре" вкл — на свечу оставляем только одно (наиболее подходящее) выделение.

### Пример (из ТЗ)
Фильтр: мин объём 180, объединение 3, перевес 60%.
На 3 уровнях: 150 bid + 50 ask → total=200 (≥180 ок).
bid_imbalance = 150/200*100 = 75; ask = 25.
max=75 > 60 → условие выполнено, доминирует BID → красим цветом bid-имбаланса.

### Права на изменение настроек
- Free: cluster search с пресетом по умолчанию БЕЗ права изменять.
- Pro: свои настройки + сохранение пресетов.
- VIP/Admin: свои настройки + пресеты + ТГ-уведомления.
- Пресет по умолчанию задаётся в админке и кэшируется; меняют его только VIP/Admin.

- Только VIP/Admin. Антиспам: дебаунс/кулдаун на одинаковые срабатывания.
