# ADMIN.md — PROCLUSTER

## Доступ
Только пользователь с ролью Admin, после авторизации. Отдельный раздел/маршрут.
Все админ-эндпоинты защищены проверкой роли на бэкенде + желательно отдельный middleware.

## Функции админки
1. Загрузка исторических данных с data.binance.vision:
   - выбор: тикер + рынок (futures/spot) + диапазон дат
   - запуск загрузки, прогресс-бар (download → parse → aggregate → insert, % и ETA)
   - идемпотентность (повтор диапазона не дублирует)
2. Добавление новых тикеров:
   - символ, рынок, priceTick, base-сжатие (futures/spot — своё), TTL хранения
   - сжатие по умолчанию на каждый ТФ
3. Мониторинг сервера:
   - CPU, RAM, диск, размер БД (ClickHouse), логи
4. Статистика пользователей:
   - кол-во по тарифам, регистрации, активность
5. Настройка возможностей по типам пользователей:
   - числовые лимиты Free/Pro/VIP (свечи, индикаторы, история)
6. Индикаторы по умолчанию:
   - какие индикаторы и с каким пресетом добавляются на график
     per тикер / рынок / ТФ
7. Пресет Cluster Search по умолчанию (кэшируется; меняют VIP/Admin на своих графиках).

## Безопасность админки (см. SECURITY.md)
- Жёсткая проверка роли на каждом запросе.
- Загрузчик истории — длительная фоновая задача, не блокировать HTTP-воркер.
- Логи действий админа (audit log) — желательно.
