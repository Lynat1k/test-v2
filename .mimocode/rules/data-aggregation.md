# Data aggregation rules (auto-loaded for market-data code)

> Это код, от которого зависит ТОЧНОСТЬ футпринта (цель — близко к ATAS / Tiger Trade).
> Любая ошибка здесь тихо искажает все графики. Детали: docs/DATA_MODEL.md.

## Единый источник правды
- Округление, маппинг цены в ценовой ряд и сжатие реализованы ОДИН раз
  и используются ВЕЗДЕ: live-агрегация, историческая загрузка, агрегация стакана.
- Запрещено дублировать логику сжатия/округления в разных местах. Только общий модуль.

## Округление объёмов (TRUNCATE до 1 знака — зафиксировано в DECISIONS.md)
- Отбрасывание, НЕ математическое округление.
- 5.1256 → 5.1 ; 5.627 → 5.6 ; 0.0125 → 0.0 ; 0.85 → 0.8
- Применять к bid/ask объёмам кластеров И к размерам в снапшотах стакана.

## Интерпретация трейда (Binance trades stream, НЕ aggTrades)
- isBuyerMaker == true  → агрессор продавец → объём в ASK/SELL сторону.
- isBuyerMaker == false → агрессор покупатель → объём в BID/BUY сторону.
- Сортировать строго по tradeId по возрастанию.
- Время — trade time, не время получения пакета.
- Не терять пакеты: при reconnect дозапросить пропуски через REST,
  проверить непрерывность tradeId.

## Сжатие тиков
- priceTick: futures BTCUSDT = 0.1$, spot BTCUSDT = 0.01$.
- base-сжатие берётся из конфигурации символа в БД/админке, НЕ хардкод.
- Уровни сжатия = base * k, до 10 уровней. Объединение рядов из base-данных.
- Цена → ценовой ряд: привязка к нижней границе ряда по шагу (priceTick * compression).

## Агрегация по таймфреймам
- Сырьё агрегируется согласно ТФ (1m раз в минуту, 1h раз в час и т.д.).
- Минимальный ТФ хранится детально; старшие — через rollup/materialized views,
  не дублируя объём. Выбранный путь зафиксирован в DECISIONS.md.

## Запись в ClickHouse
- Кластеры — ОТДЕЛЬНЫЕ строки на ценовой ряд (не вложенные массивы).
- Таблицы раздельные: clusters_futures / clusters_spot (свои TTL),
  clusters_futures_dom / clusters_spot_dom (свои TTL).
- Запросы только параметризованные, без конкатенации строк.
- Историческая загрузка идемпотентна: повтор диапазона не дублирует данные.

## Realtime
- Черновая агрегация текущей свечи — в Redis, обновляется на каждом трейде.
- Запись в ClickHouse — при закрытии интервала ТФ.
- Кэш последних 700 свечей по (symbol, market, timeframe) в Redis.

## При сомнении
- Если правило сжатия/агрегации неоднозначно для нового тикера — СПРОСИ,
  не выдумывай шаг. Сверься с docs/DATA_MODEL.md и docs/DOM_SPEC.md.
